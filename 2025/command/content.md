## 用語

#### iptables

iptablesとは一般的なLinuxに搭載されているパケットフィルタ

パケットとは、スマホのパケット通信費などでよく出てくるように、ネットワーク上を流れるデータの事だ。パケットフィルタとは、このパケットを選別して、通すものと通さないものを分けるということを意味している。

> iptables -L

```
Chain INPUT (policy DROP)
target     prot opt source      destination   
ACCEPT     all  --  anywhere    anywhere    
ACCEPT     all  --  anywhere    anywhere      state RELATED,ESTABLISHED
LOG        all  --  anywhere    anywhere      LOG level warning prefix "drop_packet: "
 
Chain FORWARD (policy DROP)
target     prot opt source      destination   
 
Chain OUTPUT (policy ACCEPT)
target     prot opt source      destination  

```

#### ルートキット（root kit）

ルートキットは、コンピューターに潜伏し不正アクセスを手助けするツールがパッケージ化されたもの

管理者権限を意味する「ルート」が攻撃対象として狙われるため、被害が大きくなるリスクが高い。直接的な被害を与えるよりも、バックドアとして機能するなど、ほかのマルウェアの活動を補佐するルートキットが多い。ルートキットには以下のようなツールが含まれる。

オペレーティングシステムの階層まで深く入り込み、自らを隠蔽する悪質なものが存在する。

ほかのマルウェアが侵入し、攻撃するのを補佐するようなツールが含まれているため、いったん感染すると被害が大きくなりやすい

ルートキットの機能

ログ改ざんツール

マルウェアの侵入を隠蔽し、マルウェアやルートキット自体の検知・駆除を遅らせる

バックドア生成ツール

ルートキットの侵入口が防がれても、ほかのマルウェアが再び侵入できるよう裏口（バックドア）を設ける

トラフィック監視ツール

ネットワークを介してやり取りされるデータを盗聴する

キーロガーツール

キーボード入力を盗み見てログを記録する。キーボード入力から個人情報や機密情報を抽出し、他のコンピューターへ送信する

また、ルートキットには二つのモードが存在し、それぞれ特徴が異なる。

ユーザーモード

感染すると、アプリケーションのプロセスを乗っ取られる、あるいはアプリケーションが使用するメモリーが上書きされてしまうといった被害につながる。

カーネルモード

OSの最下層で実行されるもので、感染するとコンピューターは完全に制御され、自由に操作されてしまうようになる。


## DLP

**DLP（Digital Light Processing）** とは、**デジタル光処理技術**の略で、主に **プロジェクターや3Dスキャナ、光造形3Dプリンタなどで用いられる光制御技術** のことを指します。

以下に、構造・原理・用途を分かりやすく整理します。

---

## 🔹 1. DLPの基本構造と原理

DLPは、**Texas Instruments（TI）社が開発した光制御方式**です。

その中心部にあるのが **DMD（Digital Micromirror Device）** と呼ばれる **微小ミラーアレイチップ** です。

### ● DMD（Digital Micromirror Device）

* 数十万～数百万個の微小な鏡（1画素あたり1枚）が格子状に配置されたチップです。
* 各ミラーは、 **電気的にON/OFFの2状態に高速で傾けられる** （通常は ±12° 程度）。
* 光源（LEDやレーザー）からの光を反射させるか外すかで、ピクセルの明暗を制御します。

---

## 🔹 2. 動作の仕組み（プロジェクターの場合）

1. **光源** （白色LEDやレーザー）からの光が DMD に照射されます。
2. 各ミラーが電気的に制御され、
   * ON（反射） → スクリーンに光を投影
   * OFF（非反射） → 光を吸収方向へ逃がす
3. このON/OFFを高速に繰り返すことで、**中間の明るさ（階調）**を再現します（PWMのように）。
4. カラー表示は以下の2方式のどちらかで実現します：
   * **単板DLP** ：1枚のDMDにRGBカラーフィルターやカラーホイールを組み合わせて時分割で表示
   * **3板DLP** ：RGBそれぞれにDMDを1枚ずつ使用して同時に表示（高画質・業務用）

---

## 🔹 3. DLPの用途

| 用途                                        | 内容                                                                      |
| ------------------------------------------- | ------------------------------------------------------------------------- |
| 🎥**プロジェクター**                  | 家庭用〜業務用まで広く利用。高コントラスト・高応答性が特徴。              |
| 🧠**3Dスキャナ**                      | パターン光を投影して、物体の表面形状を高精度に取得。                      |
| 🧬**光造形3Dプリンタ（DLPプリンタ）** | 樹脂にUV光を照射し層ごとに硬化させて立体を形成。LCD式よりも高精度・高速。 |
| 🔬**マイクロリソグラフィ**            | 半導体・バイオ分野でマスクレス露光などに使用。                            |

---

## 🔹 4. DLPの特徴（メリット・デメリット）

### ✅ メリット

* 高速応答（マイクロ秒単位）で、**動画・動的パターン投影に強い**
* 高コントラスト比（黒が締まる）
* ミラー方式のため、**経年劣化が少ない**
* 光学的に安定しており、キャリブレーションが容易

### ⚠️ デメリット

* 光学系がやや複雑（カラーホイールのノイズや虹色現象が出る場合がある）
* 単板方式では色再現が時分割でやや弱い
* 高性能DMDチップはコストが高い

---

## 🔹 5. DLPと他方式の比較

| 項目         | DLP                      | LCD（液晶）      | LCOS（反射型液晶） |
| ------------ | ------------------------ | ---------------- | ------------------ |
| 原理         | 微小ミラー反射           | 透過液晶         | 反射液晶           |
| コントラスト | 高い                     | 低〜中           | 高い               |
| 応答速度     | 非常に速い               | 遅い             | 中                 |
| 焦点ズレ     | 少ない                   | 少ない           | やや多い           |
| 用途         | プロジェクタ、3Dプリンタ | 安価プロジェクタ | 高級プロジェクタ   |

---

## 🔹 6. Pythonなどでの利用例（研究・開発向け）

DLPを制御するためのSDK（例： **TI LightCrafter SDK** ）が提供されています。

Pythonから制御したい場合には：

* **TI DLP LightCrafter API**
* **OpenCV でパターン生成 → DLPで投影 → カメラで撮像（3Dスキャン）**

  などの構成が可能です。

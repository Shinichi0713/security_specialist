1. 接続・インフラ構成（オンプレミス連携）

物理拠点とクラウドをいかに安全・高速につなぐかが問われます。
- Direct Connect / ExpressRoute: AWSやAzureなどの専用線接続サービス。BGPによる経路交換や冗長構成が狙われます。
- Transit Gateway: 複数のVPCやオンプレミス接続を中央でハブとして束ねる技術。
- Shared VPC (共有VPC): 複数のプロジェクトやアカウントで一つのネットワーク基盤を共有する構成。
- SD-WAN: 拠点間接続をソフトウェアで制御し、クラウドへの直接アクセス（ローカルブレイクアウト）を実現する技術。
 
2. クラウド固有のセキュリティ技術

「境界防御」から「アイデンティティとデータの保護」へのシフトが鮮明です。
- SASE (Secure Access Service Edge): ネットワーク（SD-WAN等）とセキュリティ（SWG, CASB, ZTNA等）をクラウド上で統合提供する枠組み。
- ゼロトラスト (Zero Trust): 「何も信頼しない」を前提に、接続ごとに認証・認可を行う考え方。
- ZTNA (Zero Trust Network Access): VPNに代わる、アプリケーション単位でのセキュアなアクセス手段。
- CASB (Cloud Access Security Broker): 従業員のクラウド利用状況を可視化し、一括してセキュリティポリシーを適用する仕組み。
- CSPM (Cloud Security Posture Management): クラウドの設定ミス（バケットの公開設定など）を自動で検知・是正する管理機能。 

3. クラウドネイティブ・運用技術

スケーラビリティと可用性の確保に関する用語です。
- Infrastructure as Code (IaC): TerraformやCloudFormationなど、ネットワーク構成をコード化して管理する手法。
- サービスメッシュ (Service Mesh): マイクロサービス間の通信（東西通信）を可視化・制御する技術（Istioなど）。
- サーバレス (AWS Lambda等): ネットワーク構成の自動抽象化と、イベント駆動型の実行環境。
- エッジコンピューティング: データの発生源に近い場所で処理を行い、クラウドへのトラフィック負荷を軽減する構成。 

4. 2026年に向けた「超」注目ワード

近年の攻撃トレンドや技術進化から、試験での登場が予想されるキーワードです。

- プロンプトインジェクションへの対策: AIを活用したクラウドサービス利用時、AIを悪用した攻撃からインフラをどう守るかという観点。
- 自律型AIセキュリティ: AIが自動で脅威を検知・遮断する運用モデルの導入。
- マルチアーキテクチャ・ガバナンス: 異なるクラウド（AWS, Azure, GCP）が混在する環境を統合管理する設計指針。

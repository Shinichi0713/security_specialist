ですが、AIの応用先としてのシステムを考える上で、ネットワークの理解が欠かせません。

普段身の回りで利用される無線通信はIEEE 802.11規格によって整備されたルールに沿って運用されています。
更に、無線通信において、OSI参照モデルの第1層「物理層（PHY）」は、PLCPとPMDという2つのサブレイヤ（副層）に分かれて機能しています。

>__物理層とは__  
>物理層（Physical Layer）は、OSI参照モデルの **第1層（最下層）** に位置し、ネットワークの「目に見える部分」や「物理的な現象」を司る層です。  
>一言でいうと、 **「0と1のデジタルデータを、電気・光・電波などの物理的な信号に変換して相手に届ける役割」** を担っています。


### 1. 物理層の2層構造

物理層を「準備担当」と「実行担当」に分けて考えると分かりやすくなります。

PLCPが「データの準備（書類の作成）」担当だとすれば、PMDは「実際に電波として叫ぶ（スピーカー）」担当といえます。

* **PLCP（Physical Layer Convergence Procedure）：準備担当**
* MAC層（上位層）から受け取ったデータに、同期用の信号やデータの長さなどの情報を付け加えます。
* これを「PLCPプロトコルデータユニット（PPDU）」と呼びます。


* **PMD（Physical Medium Dependent）：実行担当**
PLCPから受け取った「0」と「1」のデジタルデータを、空中やケーブルを通る「信号」に変換します。
デジタルデータを、実際に伝送媒体（電波や光、電気）へ乗せるための物理的な変換を行う仕組みのことです。

- PLCPが作ったデータを、実際に電波（変調）として空中に放り出します。
- 変調（Modulation）: デジタル信号を電波の波形（振幅、周波数、位相など）に変換します。Wi-Fiで使われるOFDMやQAMといった技術はここに含まれます。
- 物理的な送受信: アンテナを介して電波を発信したり、光ファイバーに光を流したりします。
- 媒体の監視: 現在、電波が使われているかどうか（エネルギー検出）を調べ、上位層に報告します。



### 2. PLCPの主な役割

PLCPがデータに付与する「PLCPヘッダ」には、受信側が正しくデータを受け取るための重要な情報が含まれています。

1. **同期（プリアンブル）:**
「今からデータを送りますよ！」という合図を送り、受信側のルーターとタイミングを合わせます。
2. **通信パラメータの伝達:**
「このデータはどのくらいの速度（変調方式）で送るのか」「データの長さはどれくらいか」という情報を伝えます。
3. **キャリアセンス（仮想的）:**
電波が使われている時間を他のデバイスに知らせ、衝突を防ぐ手助けをします。


### 3. なぜPLCPが必要なのか？

無線通信は、有線（UTPケーブルなど）に比べてノイズの影響を受けやすく、通信環境が刻一刻と変化します。

もしPLCPがなければ、受信側は「いつデータが始まったのか」「どのくらいの速さで読み取ればいいのか」が分からず、電波の波をデータとして復元することができません。PLCPが **「データの取扱説明書」** を先頭に付けてくれるおかげで、安定した通信が可能になります。


### 4. まとめ
無線通信における物理層はPLCPとPMDによって構成されています。
2つのサブレイヤの働きは以下の通りと言えます。

- PLCP: MAC層と物理層を「適合（Convergence）」させるための論理的な手続き。
- PMD: 実際の「物理媒体（Medium）」に「依存（Dependent）」した電気的・光学的な処理。
